# 报告进度工具提示词

## 功能说明

`multi-agent::report` 工具用于子智能体向主智能体正式汇报任务进度和结果。

## 参数说明

- **progress**：必填，任务完成进度（0-100 的百分比）
- **message**：必填，汇报消息内容
- **status**：必填，任务状态（in_progress / completed / failed / blocked）

## 使用场景

- 阶段性任务完成后向主智能体汇报
- 遇到无法解决的困难时请求帮助
- 任务执行过程中定期更新进度
- 任务完成后提交最终结果

## 使用示例

```json
{
  "progress": 75,
  "message": "已完成核心模块开发，正在进行单元测试",
  "status": "in_progress"
}
```

```json
{
  "progress": 100,
  "message": "所有任务已完成，代码已提交到仓库",
  "status": "completed"
}
```

```json
{
  "progress": 30,
  "message": "遇到技术难题：无法连接到数据库，请协助排查",
  "status": "blocked"
}
```

## 状态说明

1. **in_progress**：任务正在进行中
2. **completed**：任务已成功完成
3. **failed**：任务执行失败
4. **blocked**：任务受阻，需要主智能体协助

## 汇报规范

- **及时汇报**：任务有重大进展或遇到问题时及时汇报
- **准确进度**：提供准确的进度百分比
- **清晰描述**：消息内容要清晰描述当前状态
- **问题说明**：遇到问题时详细说明原因和已尝试的解决方案

## 注意事项

1. **报告频率**：避免过于频繁的报告，每完成一个阶段后再报告
2. **信息完整**：报告内容应包含足够信息供主智能体决策
3. **阻塞处理**：遇到阻塞时先说明问题，再说明需要的帮助
